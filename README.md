# École — публикация через GitHub Pages

Эта папка содержит готовую статическую версию сайта.

## Что находится внутри

- `index.html` — страницы и разметка сайта;
- `styles.css` — оформление;
- `app.js` — интерактивность;
- `vendor/gsap.min.js` — локальная библиотека анимаций.

## Как впервые опубликовать сайт

1. Зарегистрируйтесь на https://github.com.
2. Нажмите **New repository**.
3. Назовите репозиторий, например `ecole`.
4. Выберите **Public** и нажмите **Create repository**.
5. На странице репозитория нажмите **uploading an existing file** или **Add file → Upload files**.
6. Перетащите в окно GitHub содержимое этой папки. Важно: файл `index.html` должен лежать в корне репозитория, а не внутри дополнительной папки.
7. Внизу напишите `Первая версия сайта` и нажмите **Commit changes**.
8. Откройте **Settings → Pages**.
9. В разделе **Build and deployment** выберите **Deploy from a branch**.
10. Выберите ветку `main`, папку `/ (root)` и нажмите **Save**.
11. Через несколько минут GitHub покажет ссылку вида `https://ВАШЕ-ИМЯ.github.io/ecole/`.

## Как обновлять сайт без командной строки

1. Откройте репозиторий на GitHub.
2. Нажмите **Add file → Upload files**.
3. Перетащите обновлённые файлы из этой папки.
4. Подтвердите замену файлов и нажмите **Commit changes**.
5. GitHub Pages автоматически опубликует новую версию через несколько минут.

Git хранит версии файлов сайта. Он не является базой данных и не должен хранить профили, пароли, учеников, расписание или сообщения.

## Где будут храниться настоящие данные

Для регистрации и личных кабинетов нужно отдельно подключить Supabase:

- Supabase Auth — регистрация и вход;
- Supabase Database — профили, ученики, уроки и задания;
- Supabase Storage — аватары и учебные файлы.

Пока Supabase не подключён, сайт является демонстрационным прототипом. Профиль сохраняется только в браузере текущего устройства.

## Важно

Никогда не загружайте на GitHub пароли, секретные ключи, списки учеников или другие личные данные.
